Rules for AI coding:

- Read docs/project_brief.md and docs/database_schema.md before coding.
- Do not create new database tables.
- Do not rename existing tables or columns.
- Use Supabase Flutter client.
- Keep authentication logic separate from business features.
- UI text must be Romanian.
- Code identifiers and filenames should be English.
- Use feature-first architecture.
- Generate complete files only.
- Do not put business logic in main.dart.
- Do not fetch data inside build().
- Use ListView.builder for lists.
- Use Supabase views for dashboard and workshop details.
- Keep UI responsive for desktop, tablet and mobile.
- Do not use old schema tables.
